Web Chat App Flow
Claude Code tmux bridge를 웹 채팅앱에 붙이는 흐름도

브라우저 채팅창은 CLI stream 한 번을 대화 turn 하나로 다룬다

웹앱은 사용자의 메시지를 백엔드로 보내고, 백엔드는 `claude_tmux_control.py stream`을 실행해서 JSONL event를 SSE/WebSocket으로 그대로 중계한다.

one browser conversation = one session_id one submit = one stream process done unlocks input metrics follows done

핵심 계약

`session_id`가 없으면 CLI가 UUID를 만들고, 첫 JSONL event부터 그 값을 포함한다.
같은 대화창은 다음 turn부터 같은 `session_id`를 다시 보낸다.
tmux가 사라져도 기존 state/transcript가 있으면 `--resume`으로 복구한다.

End-to-End Flow

각 상자는 한 시스템 경계를 나타냅니다. 브라우저는 tmux 세부사항을 몰라도 됩니다.

runtime path
Browser Chat UI user message submit, input disabled session_id optional
App Backend spawn CLI and bridge stdout JSONL to client SSE or WebSocket
ctc stream ensure session, send prompt, tail transcript one turn
tmux + Claude Code inline prompt on start/resume, paste prompt on active tmux terminal UI
Claude Transcript JSONL source for answer, tools, usage structured source

Command Builder

웹 서버가 실제로 실행할 high-level stream command를 만들어 봅니다.

copy target

Claude Code는 기본적으로 --dangerously-skip-permissions로 시작한다. 변경이 필요하면 backend의 안전한 설정을 --claude-args "--permission-mode ..."로 매핑하고, browser 입력을 그대로 전달하지 않는다.

Backend Responsibilities

        

          
            
단계해야 할 일

          
          
            
request대화창이 가진 `session_id`를 넘긴다. 새 대화면 생략한다.

            
spawn`stream --cwd ...`를 실행한다. account별 token은 `--oauth-token-env`로 선택한다.

            
relaystdout JSONL을 한 줄씩 parse해서 SSE/WebSocket message로 전달한다.

            
persist첫 event의 `session_id`를 conversation record에 저장한다.

            
unlock`done` 수신 후 입력창을 다시 활성화한다. 바로 뒤 `metrics`를 usage 패널에 반영한다.

            
metricsCLI final `metrics`의 `elapsed_ms`, `cost.turn_usd`, `cost.session_usd`를 usage 패널에 반영한다.

          
        

      


      

        
Session Decision Table

        

          
            
상황CLI 동작UI 의미

          
          
            
session_id 없음UUID 생성, `claude --session-id ... -- <prompt>`새 대화 시작

            
active tmux 있음기존 pane에 prompt 전송대화 이어가기

            
inactive + state/transcript 있음`claude --resume ... -- <prompt>`복구 후 이어가기

            
cwd mismatch`session_cwd_mismatch` error잘못된 대화 연결 차단

            
active turn working`turn_in_progress` error중복 전송 차단

            
user cancel`ctc cancel "$SESSION_ID"` sends `Escape`취소 요청 후 `last`/`attach`로 완료까지 수신

            
tool cancel marker`User rejected tool use` + interrupt markerfinal answer 없이도 `done`/`metrics`로 turn 종료

            
timeout turn already donestate-only finalize 후 새 prompt 전송수동 attach 없이 다음 입력 진행

            
completed turn replay`ctc last "$SESSION_ID" --last N` 또는 `replay`이탈한 채팅창에 최근 JSONL event 재전송

            
latest turn still active`last`/`replay --last N`이 active_turn에 attach완료 turn replay 후 active turn을 물고 `done`/`metrics` 수신

          
        

      

    


    

      

        

          
Guards And Gaps

          
현재 구현 기준으로 믿어도 되는 것과 앱 서버가 보완해야 하는 것을 구분합니다.

        

        implementation boundary
      


      

        

          
Implemented

          
UUID validation, cwd fail-closed, send_lock, active_turn, cancel via Escape, transcript sessionId filtering, timeout auto-finalize, offset tailing, completed turn replay, done + metrics.

        

        

          
CLI Metrics

          
elapsed_ms, turn cost, session cumulative cost는 CLI final `metrics`에서 받는다. UI-specific retry policy는 앱 서버가 정한다.
