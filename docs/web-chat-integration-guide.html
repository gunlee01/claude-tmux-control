Claude Code interactive session을 웹 채팅 UI로 붙이는 운영 지도

웹 대화창 하나는 bridge session_id 하나를 계속 쓴다

웹앱이 UUID를 넘기거나 CLI가 UUID를 만들고 `ctc-csess-<session_id>` tmux session을 띄운다. 첫 시작은 `--session-id`, 복구는 `--resume`으로 Claude Code session을 연결한다.

tmux session reuse JSONL stream tool call visibility turn metrics

TL;DR

  • 새 대화는 앱 서버가 UUID `session_id`를 넘기는 방식을 권장한다. 생략하면 CLI가 만들고 첫 event에 포함한다.
  • 같은 웹 대화창은 같은 `ctc-csess-<session_id>` tmux session을 사용한다.
  • tmux가 죽어 있으면 같은 `session_id`로 `claude --resume`을 실행한다.
  • UI 진행 정보는 `stream`의 `thinking`, `tool_use`, `tool_result`, `assistant_text`, `done`을 line-by-line으로 읽는다.
  • elapsed time, token, turn cost, session cumulative cost는 final `metrics` event에서 받는다. context는 transcript에 있을 때만 받는다.
  • turn/session cost는 CLI가 `claude_pricing.json`과 completed turn records 기준으로 계산한다.

Session Identity

canonical id는 앱 서버가 넘기거나 CLI가 생성한 UUID다. tmux session name과 Claude Code session id는 이 값을 기준으로 맞춘다.

mapping rule

session_id

앱 서버가 생성해 넘기거나 CLI가 생략 시 생성한 UUID. 클라이언트는 다음 요청에 이 값을 그대로 돌려준다.

tmux_session

`ctc-csess-<session_id>` 형식. CLI가 실제로 제어하는 tmux key다.

Claude sessionId

첫 시작 때 `--session-id <session_id>`로 맞춘 Claude Code 내부 id다.

권장 형식 canonical hyphenated UUID string을 사용한다. CLI 생성값은 UUID v4이고, 대문자 입력은 lowercase canonical form으로 정규화한다. client 생성값도 v4를 권장하지만 강제하지 않는다. `:`는 tmux target separator라서 session name에 넣지 않는다.
재사용 기준 `tmux has-session -t ctc-csess-<session_id>`가 active면 같은 Claude Code process를 계속 쓴다.
매핑 저장소 별도 매핑 DB 없이 tmux session name에서 복원한다. transcript `sessionId`는 검증 metadata다.

Command Builder

session id, 프로젝트 경로, 계정 env를 넣으면 웹 서버가 호출할 고수준 stream command를 만든다.

interactive
웹/외부 프로그램 `stream --cwd ... [--session-id ...]`, `stream --attach --session-id ...`, `cancel`, `last`/`replay`, `ask`, `info`, `list`, `reap`을 사용한다.
Cancel 조건 `cancel`은 내부 tmux pane에 `Escape`를 보내고 JSON을 반환한다. tool 취소는 final answer 없이 `done`/`metrics`로 닫힐 수 있다.
Attach 조건 `attach`는 완료된 과거 turn 조회가 아니다. `active_turn`이 남은 active/timeout/interrupted turn에만 다시 붙는다.
Replay 조건 `last`/`replay`는 완료된 최근 turn을 JSONL로 다시 출력한다. 마지막 turn이 진행 중이면 완료 turn replay 후 현재 `active_turn`에 attach한다.
저수준 디버깅 `start`, `send`, `answer`, `turn`, `events`, `kill`은 tmux session name을 받는다. bridge UUID를 직접 넣지 않는다.
Process stop `kill <tmux_session>`은 tmux/Claude process를 멈춘다. high-level bridge state나 transcript를 삭제하지 않는다.
웹 UI 답변 `answer "$SESSION_ID"`를 호출하지 않는다. 일반 완료는 `stream`의 `done.answer` 또는 `ask_result.answer`를 사용한다. 취소된 turn은 answer 없이 `done`/`metrics`만 올 수 있다.
환경 주입 새 tmux session을 만들 때 `/.ctc.env`, `--env-file`, `--env NAME`을 Claude process에 전달할 수 있다. 이미 실행 중인 session에는 반영되지 않는다.

Operational Decisions

          
웹 서비스로 붙일 때 가장 먼저 정해야 하는 운영 정책이다.

        

        checklist
      


      

        

          
            

              주제
              권장 정책
              CLI 기준
            

          
          
            

              session id
              client 값도 UUID로 검증하고, 기존 state의 canonical cwd와 요청 cwd가 다르면 거절한다.
              `session_cwd_mismatch`로 fail closed한다.
            

            

              동시 입력
              `send_lock`은 prompt 전송만 보호하고, 긴 stream 동안은 `active_turn`이 새 prompt를 막는다.
              이전 timeout/interrupted turn이 완료 확인되면 다음 prompt 전에 state-only finalize한다.
            

            

              완료 판단
              `done` event 전에는 입력창을 비활성화한다.
              `stream`은 transcript ready + tmux ready + idle window를 조합한다.
            

            

              툴콜 표시
              `tool_use`는 실행 시작, `tool_result`는 결과로 렌더링한다.
              둘 다 완료 신호가 아니다.
            

            

              오래된 session
              web-only 운영은 `reap --idle-seconds 1800 --prefix ctc-csess-`를 주기 실행한다.
              `ctc-` prefix는 low-level `ctc-*` session도 포함할 수 있다. `active_turn`이 남아 있고 ready가 아니면 skip한다.
            

            

              비용
              CLI final `metrics.cost`를 표시하고, 필요하면 앱 서버에서 별도 회계값과 대조한다.
              `turn_usd`와 `session_usd`를 제공한다.
            

          
        

      


      

        

          
현재 gap

          
더 세분화된 inactive/starting 상태와 transcript rotation follow 고도화는 남은 작업이다. `ensure`는 고수준 stream 내부 단계로 둔다.

        

        

          
local cursor storage

          
`~/.cache/claude-tmux-control/sessions/<session_id>.json`에 transcript offset, turn cursor, active owner, event id, usage totals를 저장해서 전체 JSONL 재읽기를 피한다. completed turn state는 최신 200개만 보관하므로 `session_usd`와 totals는 이 retained window 기준이다.
