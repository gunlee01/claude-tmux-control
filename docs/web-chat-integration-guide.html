Claude Code interactive session을 웹 채팅 UI로 붙이는 운영 지도

웹 대화창 하나는 bridge session_id 하나를 계속 쓴다

bridge가 UUID를 만들고 `ctc-csess-<session_id>` tmux session을 띄운다. 첫 시작은 `--session-id`, 복구는 `--resume`으로 Claude Code session을 연결한다.

TL;DR

  • 새 대화는 bridge가 UUID `session_id`를 만들고 클라이언트에 돌려준다.
  • 같은 웹 대화창은 같은 `ctc-csess-<session_id>` tmux session을 사용한다.
  • tmux가 죽어 있으면 같은 `session_id`로 `claude --resume`을 실행한다.
  • UI 진행 정보는 `stream`의 `thinking`, `tool_use`, `tool_result`, `assistant_text`, `done`을 line-by-line으로 읽는다.
  • elapsed time은 앱 서버가 직접 측정하고, token/context는 raw transcript event에서 추출한다.
  • cost는 CLI가 계산하지 않는다. 앱 서버의 pricing table version 기준으로 estimated cost를 계산한다.

Session Identity

canonical id는 bridge가 만든 UUID다. tmux session name과 Claude Code session id는 이 값을 기준으로 맞춘다.

mapping rule

session_id

bridge가 생성한 UUID. 클라이언트는 다음 요청에 이 값을 그대로 돌려준다.

tmux_session

`ctc-csess-<session_id>` 형식. CLI가 실제로 제어하는 tmux key다.

Claude sessionId

첫 시작 때 `--session-id <session_id>`로 맞춘 Claude Code 내부 id다.

권장 형식 UUID v4를 사용한다. `:`는 tmux target separator라서 session name에 넣지 않는다.
재사용 기준 `tmux has-session -t ctc-csess-<session_id>`가 active면 같은 Claude Code process를 계속 쓴다.
매핑 저장소 별도 매핑 DB 없이 tmux session name에서 복원한다. transcript `sessionId`는 검증 metadata다.

Operational Decisions

          
웹 서비스로 붙일 때 가장 먼저 정해야 하는 운영 정책이다.

        

        checklist
      


      

        

          
            

              주제
              권장 정책
              CLI 기준
            

          
          
            

              동시 입력
              같은 `session_id`는 반드시 session lock을 잡고 한 turn씩 처리한다.
              현재 CLI는 lock을 내장하지 않는다.
            

            

              완료 판단
              `done` event 전에는 입력창을 비활성화한다.
              `stream`은 transcript ready + tmux ready + idle window를 조합한다.
            

            

              툴콜 표시
              `tool_use`는 실행 시작, `tool_result`는 결과로 렌더링한다.
              둘 다 완료 신호가 아니다.
            

            

              오래된 session
              scheduler에서 `reap --idle-seconds 1800 --prefix ctc-`를 주기 실행한다.
              `reap`은 daemon이 아니라 scan 후 종료한다.
            

            

              비용
              앱 서버가 raw usage와 pricing table로 estimated cost를 계산한다.
              CLI는 cost를 보장하지 않는다.
            

          
        

      


      

        

          
현재 gap

          
`ensure`, `ask`, `info`, `stats`, `stream done usage summary`, session lock은 아직 future work다. `ensure`는 UUID 생성, `--session-id`, `--resume` 판단을 담당해야 한다.
